# 项目演进

早期详细开发记录已经清理。本文件只保留项目演进脉络，避免把过期状态和内部过程文档放进公开仓库。

## 1. 个人投研原型

项目最初目标是做一个个人 AI 投研助手：输入股票和上下文，产出结构化分析，辅助日常观察和交易复盘。

## 2. 数据层搭建

随后开始补数据基础设施，包括多数据源、缓存、本地历史数据、A 股相关字段、新闻公告研报等。

这个阶段的重点是让分析不要依赖单一数据源。

## 3. Agent 分析链路

在数据层基础上，引入 TradingAgents 风格的多角色分析。项目开始区分市场、情绪、新闻、基本面、政策、资金、风险、最终决策等阶段。

同时增加 LLM provider 抽象、mock mode、质量检查和信号工具。

## 4. 组合和日常运行

项目从“单次分析”转向“长期运行”：

- 增加组合和持仓；
- 增加每日交易日志；
- 增加交易费用计算；
- 增加后台任务和调度；
- 增加分析记录落库。

## 5. 知识库系统

为了让历史分析能被复用，项目加入 raw/wiki/derived 三层知识系统。

raw 保存原始材料，wiki 生成可读页面，derived 支持后续检索和结构化索引。

## 6. Web 工作台

前端迁移到 React + FastAPI，形成可操作的本地工作台，包括个股、分析、组合、交易日志、原始材料、Wiki、板块、设置等页面。

## 7. GitHub 化整理

当前阶段是把项目从开发过程状态整理成可以公开的中级项目：

- 删除过期历史文档；
- 删除旧运行产物；
- 保留源码、测试和 schema；
- 补齐中文文档；
- 强化 `.gitignore`；
- 确保个人数据默认不会被提交。
