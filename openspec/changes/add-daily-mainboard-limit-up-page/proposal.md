## 为什么

当前系统已有股票行情详情能力，但缺少一个面向每日盘后复盘的入口，帮助用户快速查看 A 股主板当日涨停股票并继续进入个股行情分析。新增该页面可以让用户从“今日涨停池”直接筛选、浏览并跳转到具体股票行情，提升选股和复盘效率。

## 变更内容

- 新增“每日 A 股主板涨停股列表筛选”页面，支持按交易日查看沪深主板涨停股票列表。
- 新增主板涨停股数据查询能力，返回股票代码、名称、市场、最新价、涨跌幅、成交额、换手率、封单/首次封板等可用字段。
- 列表提供基础筛选能力，至少支持交易日、市场范围（沪市主板/深市主板/全部主板）和股票代码/名称搜索。
- 列表中每只股票可点击跳转到现有个股行情详情页面或行情路由。
- 增加空状态、加载状态、错误提示和数据刷新入口，保证数据不可用时页面可理解。

## 功能 (Capabilities)

### 新增功能
- `daily-mainboard-limit-up`: 覆盖按交易日筛选 A 股主板涨停股票列表，并从列表跳转到个股行情详情的用户能力。

## 影响

- 前端：新增涨停股列表页面、导航入口、API client 方法，以及跳转到现有行情详情的链接。
- 后端：新增或扩展行情/市场数据 API，用于获取指定交易日 A 股主板涨停股票列表。
- 数据层：复用现有行情采集器能力；如现有数据源无法直接提供涨停列表，需要封装涨跌停筛选逻辑或接入现有 A 股行情数据源的涨停池接口。
- 测试：增加 API 查询、主板过滤、前端渲染/筛选和跳转行为的测试或验证步骤。
