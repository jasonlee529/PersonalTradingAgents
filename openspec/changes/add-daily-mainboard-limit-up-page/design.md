## 上下文

现有系统提供 `/stocks/{symbol}/quote`、`/stocks/{symbol}/kline`、`/stocks/{symbol}/snapshot` 等个股行情接口，前端 `/stock` 页面通过 `StockDetailPage` 展示所选股票的行情、K 线、基本面、技术指标和资讯。当前股票详情页依赖持仓列表选择股票，尚没有一个面向每日复盘的“主板涨停股池”入口。

本变更需要跨前端页面、API client、后端股票路由和数据采集层。实现应尽量复用现有 `DataCollector`、缓存和股票详情展示，不引入新的持久化模型，除非数据源能力不足时需要轻量封装。

## 目标 / 非目标

**目标：**

- 提供一个新页面用于查看指定交易日的 A 股主板涨停股票列表。
- 支持按交易日、主板市场范围和代码/名称关键字筛选。
- 每条股票记录都能跳转到现有行情详情能力，并让详情页加载对应股票。
- 后端提供结构化 API，统一返回前端需要的涨停列表字段。
- 对无数据、加载中、错误和刷新场景提供清晰交互。

**非目标：**

- 不实现创业板、科创板、北交所等非主板涨停股筛选。
- 不新增交易策略、自动买卖、涨停原因智能归因或推荐排序。
- 不要求长期保存历史涨停池；优先查询实时/历史行情数据源并使用现有缓存。
- 不重做现有个股行情详情页面，只补齐从涨停列表进入详情的路径。

## 决策

1. **新增独立页面与路由**
   - 新增前端页面（建议路由 `/limit-up` 或 `/stocks/limit-up`），通过侧边栏新增“涨停池”入口。
   - 理由：该能力是独立的每日复盘入口，不应混入持仓或个股详情页面。
   - 替代方案：在首页或股票详情页嵌入模块；缺点是筛选、表格和未来扩展空间不足。

2. **后端在股票域提供列表 API**
   - 在现有 `src/api/routers/stocks.py` 下新增类似 `GET /stocks/limit-up` 的集合端点，查询参数包含 `trade_date`、`market`、`q`、`limit`、`offset`。
   - 返回 Pydantic 响应模型，例如 `LimitUpStockItem` 和 `LimitUpStockListResponse`，字段包含 `symbol`、`name`、`market`、`trade_date`、`price`、`change_pct`、`turnover`、`volume`、`turnover_rate`、`first_limit_up_time`、`last_limit_up_time`、`seal_amount` 等可空字段。
   - 理由：该 API 属于股票行情集合查询，放在 stocks 路由可复用现有认证、依赖注入和 collector。
   - 替代方案：新增独立 `market` 路由；更清晰但当前项目股票相关 API 已集中在 stocks，新增路由成本更高。

3. **数据采集层封装“涨停池”能力**
   - 在 `DataCollector` 增加 `get_limit_up_stocks(trade_date, market)` 方法，内部优先调用现有国内行情源中可支持涨停池的数据源方法（如 Eastmoney/同花顺封装），并统一字段名。
   - 主板过滤在采集层或 API 层统一执行：沪市主板通常为 `60xxxx`，深市主板通常为 `00xxxx`；排除 `30xxxx`、`68xxxx`、`8/4/92xxxx` 等非主板代码。
   - 理由：避免前端承担数据源细节，也方便缓存和测试。
   - 替代方案：前端拉全市场数据后自行过滤；会增加传输量且过滤规则不可复用。

4. **股票详情跳转复用现有 `/stock` 页面**
   - 列表行或股票代码链接跳转到 `/stock?symbol=<symbol>`。
   - 同时让 `StockDetailPage` 读取 URL query 中的 `symbol`，在存在该参数时调用 `setSelectedSymbol(symbol)` 并加载 snapshot；这样无需强制该股票已在持仓中。
   - 理由：复用现有详情页组件和 API，避免新增重复详情页面。
   - 替代方案：新增 `/stock/:symbol` 路由；语义更直接，但改动面略大，可作为后续增强。

5. **前端筛选采用服务端查询 + 本地关键字兜底**
   - 交易日、市场范围触发后端重新查询；关键字 `q` 传给 API，如后端暂不支持则前端也可对当前结果做本地过滤。
   - 理由：交易日和市场维度影响数据集，适合由后端统一处理；关键字过滤成本低，可提升交互响应。

## 风险 / 权衡

- **数据源字段不稳定或当日盘中数据延迟** → 响应模型字段允许部分为空，前端用 `-` 展示缺失值，并显示数据日期/刷新按钮。
- **涨停判定因 ST、不同板块涨跌幅限制而复杂** → 本期范围限定 A 股主板；优先使用数据源的涨停池接口，不自行用 10% 简化计算作为唯一依据。
- **股票详情页当前依赖持仓选择** → 增加 URL symbol 支持时要保证原持仓选择逻辑不被破坏：无 query 时仍默认使用持仓首项，有 query 时优先展示 query 股票。
- **OpenSpec 配置存在 YAML 缩进解析告警** → 当前 CLI 仍能生成和读取变更状态，但后续实现前应考虑修复 `openspec/config.yaml` 的缩进问题，避免严格校验失败。
- **接口路径与动态路由冲突** → 如果在 `/stocks/{symbol}/...` 同文件新增集合端点，应将 `/limit-up` 这类静态路径定义在动态 `{symbol}` 路由之前，或选择不会被动态路由吞掉的路径。
