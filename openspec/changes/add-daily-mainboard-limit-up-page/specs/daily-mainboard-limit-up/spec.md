## 新增需求

### 需求:按交易日查询主板涨停股
系统必须允许用户按交易日查询 A 股主板涨停股票列表，并且结果必须只包含沪市主板和深市主板股票。

#### 场景:查询指定交易日
- **当** 用户选择一个有效交易日并打开涨停股列表
- **那么** 系统返回该交易日的 A 股主板涨停股票列表

#### 场景:排除非主板股票
- **当** 数据源返回创业板、科创板、北交所或其他非主板股票
- **那么** 系统禁止将这些股票展示在主板涨停股列表中

### 需求:筛选涨停股列表
系统必须提供交易日、市场范围和代码/名称关键字筛选能力。

#### 场景:按市场范围筛选
- **当** 用户选择“沪市主板”“深市主板”或“全部主板”
- **那么** 系统必须只展示符合所选市场范围的涨停股票

#### 场景:按关键字筛选
- **当** 用户输入股票代码或股票名称关键字
- **那么** 系统必须只展示代码或名称匹配关键字的涨停股票

### 需求:展示涨停股行情字段
系统必须在列表中展示每只涨停股票的核心行情字段，包括股票代码、股票名称、市场、交易日、最新价、涨跌幅、成交量或成交额，并在数据源提供时展示换手率、首次封板时间、最后封板时间和封单金额。

#### 场景:字段完整时展示详情
- **当** 数据源返回某只涨停股票的完整行情字段
- **那么** 系统必须在该股票行中展示所有可用字段

#### 场景:部分字段缺失时展示占位
- **当** 数据源未返回某个可选字段
- **那么** 系统必须使用清晰占位值展示该字段，并禁止因字段缺失导致列表加载失败

### 需求:跳转到个股行情详情
系统必须允许用户从涨停股列表中的每只股票跳转到该股票的行情详情。

#### 场景:点击股票代码或名称
- **当** 用户点击列表中某只股票的代码或名称
- **那么** 系统必须打开该股票的行情详情页面并加载对应股票的数据

#### 场景:股票不在持仓中
- **当** 用户点击的涨停股票不在当前持仓列表中
- **那么** 系统仍必须展示该股票的行情详情，而不是要求用户先添加持仓

### 需求:处理加载空态和错误
系统必须为涨停股列表提供加载状态、空结果状态、错误提示和手动刷新能力。

#### 场景:列表加载中
- **当** 系统正在获取涨停股列表
- **那么** 页面必须展示加载状态

#### 场景:没有匹配结果
- **当** 指定交易日和筛选条件下没有涨停股票
- **那么** 页面必须展示明确的空结果提示

#### 场景:获取数据失败
- **当** 后端或数据源获取涨停股列表失败
- **那么** 页面必须展示错误提示，并提供重新加载入口
